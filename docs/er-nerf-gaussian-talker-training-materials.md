# ER-NeRF 与 GaussianTalker 训练素材方案

## 1. 方案结论

当前数字人 PoC 优先使用以下两套模型进行对比：

- ER-NeRF：验证现有实时直播链路、音频驱动和低延迟推理能力。
- GaussianTalker：验证 Gaussian Splatting 路线的画面清晰度、身份保持和渲染性能。

两套模型必须使用同一段原始人物视频、相同的裁剪范围和帧率进行训练，避免素材差异影响对比结果。第一轮素材优先选择 GeneFace 官方人物视频，不直接使用来源不明的网络口播视频。

首选人物为 `May.mp4`。GeneFace 官方同时提供了该人物的原始视频、预处理数据和 v1.1.0 模型包，便于先验证素材质量和预处理流程，再将同一原始视频分别输入 ER-NeRF 与 GaussianTalker。

第一轮建议按 `May -> Jae-in -> Shaheen` 的顺序测试。不要一次训练全部人物，应先通过 `May` 确认预处理、训练、推理和直播接入链路正常，再增加复测人物。

### 2.3 为什么优先使用 May

- GeneFace README 的快速开始流程直接使用 `May.mp4`。
- 官方素材目录提供 `data/raw/videos/May.mp4` 和处理后的 `trainval_dataset.npy`。
- v1.1.0 Release 提供 `May.zip`，可以辅助核对官方处理结果和模型输出。
- 原始视频可重新按 ER-NeRF 与 GaussianTalker 的要求统一预处理，不依赖 GeneFace 的二进制数据格式。
- 出现效果问题时，可以更容易判断问题来自原视频、预处理、模型训练还是实时推理链路。

## 3. 素材初筛标准

下载后需要先检查原视频，不符合要求的片段不进入训练：

- 单人、正面或轻微侧脸、固定机位、连续口播。
- 建议有效时长 1 至 5 分钟，第一轮使用 90 秒和 150 秒两个版本。
- 人脸清晰，眼睛、嘴唇、牙齿和下巴没有明显压缩块。
- 光照、曝光和白平衡稳定，避免频繁闪烁或强烈阴影变化。
- 背景稳定，人物与摄像机距离基本不变。
- 允许自然眨眼和小幅点头，但避免大角度转头、长期闭眼和快速晃动。
- 避免手遮挡脸部、麦克风遮挡嘴部、眼镜强反光和头发频繁遮住嘴部。
- 音频应连续清晰，尽量没有背景音乐、混响、多人说话和明显降噪失真。

这两套模型的重点仍是头部和上半身区域。训练视频即使包含完整身体，也不会自动获得可指定的全身肢体动作能力。

## 4. 统一预处理规格

为了公平比较 ER-NeRF 和 GaussianTalker，每个人物只维护一份标准化母版，再从母版生成各模型需要的数据：

| 项目 | 建议规格 |
| --- | --- |
| 视频编码 | H.264，progressive，`yuv420p` |
| 帧率 | 恒定 25 FPS |
| 训练画幅 | 以脸部为中心的 512 x 512 |
| 训练时长 | 90 秒和 150 秒各一版 |
| 音频 | 单声道 PCM/WAV 中间文件，采样率按模型预处理脚本要求生成 |
| 画面范围 | 完整保留头部、下巴、颈部和肩膀，避免裁掉发梢或下巴 |
| 数据复用 | 两个模型使用同一时间范围、同一裁剪结果和同一音轨 |

预处理步骤：

1. 下载官方原始人物视频并记录文件哈希、分辨率、帧率、时长和编码信息。
2. 人工检查整段视频，确定连续、无遮挡、光照稳定的有效时间范围。
3. 转换为恒定 25 FPS，并消除旋转元数据、隔行扫描和可变帧率影响。
4. 根据整段人脸轨迹确定统一裁剪框，输出 512 x 512 标准化母版。
5. 从相同起点分别导出 90 秒和 150 秒训练版本。
6. 分别运行 ER-NeRF 和 GaussianTalker 的预处理脚本，不在模型分支中重新裁剪源视频。
7. 保存预处理日志和失败帧，便于定位人脸关键点、语义分割或相机估计问题。

## 5. 第一轮实验安排

### 阶段 A：May 流程验证

- 使用 `May.mp4` 生成 90 秒标准化训练视频。
- 分别完成 ER-NeRF 和 GaussianTalker 的预处理、训练与离线推理。
- 使用相同的中文和英文测试音频驱动两个模型。
- 确认离线效果可接受后，再接入现有实时直播链路。

### 阶段 B：训练时长比较

- 使用同一个人物比较 90 秒与 150 秒训练数据。
- 观察增加数据后，嘴部细节、头部稳定性、未见音频泛化和训练耗时是否明显改善。
- 如果 150 秒版本收益有限，后续人物统一使用 90 秒以降低训练成本。

### 阶段 C：跨人物复测

- 依次使用 `Jae-in.mp4` 和 `Shaheen.mp4` 重复相同流程。
- 不针对单个人物临时修改裁剪范围或测试音频，保持实验条件一致。
- 至少完成一名男性和一名女性人物的结果对比后，再确定默认模型。

## 6. 对比指标

每次训练记录以下数据：

- 预处理耗时、训练耗时、峰值显存和模型文件大小。
- 离线渲染 FPS、实时推理 FPS、首帧延迟和音频到画面端到端延迟。
- 中文与英文口型同步，重点观察闭唇音、爆破音、牙齿和舌头细节。
- 身份保持，包括脸型、眼睛、发型、肤色和肩颈区域。
- 时序稳定性，包括头部抖动、嘴部闪烁、背景漂移和肩膀撕裂。
- 长时间直播稳定性，包括显存增长、音画漂移、卡顿和服务恢复能力。

第一轮不只比较单帧清晰度。实时直播场景应优先保证时序稳定、口型同步、低延迟和持续运行能力。

## 7. 素材授权边界

GeneFace 项目代码或数据处理流程的开源许可，不等同于人物肖像、声音和原视频可以直接商用。上述官方人物素材仅用于内部 PoC、技术验证和模型对比，不用于公开直播、营销内容或商业交付。

正式上线时，应由员工、签约演员或客户本人重新录制训练视频，并签署至少覆盖以下内容的授权：

- 肖像和声音采集。
- AI 模型训练、推理和数字人生成。
- 直播、宣传和商业使用范围。
- 使用期限、平台范围、数据保存和删除方式。
- 模型及衍生素材的归属、撤回和终止处理机制。
